How It Works

Simple workflow, powerful results

1

Question Posted

User posts a question in any monitored Slack channel. The bot detects it instantly and saves it to the database with metadata.

2

Grace Period

Configurable waiting period allows natural responses without triggering immediate alerts. Respects conversation flow.

3

Multi-Tier Escalation

If unanswered, questions escalate through configured levels: user groups, specific users, or channels. Each tier has independent timing and targets.

4

Resolution

Once answered (via emoji reaction, thread reply, or both depending on mode), escalation stops and question is marked resolved with metrics captured.

Available Commands

Complete control with simple slash commands

/qr-setup

Comprehensive setup wizard for first-time configuration. Walks through escalation timing, target selection, and answer detection mode with validation.

/qr-config

Opens interactive modal to configure workspace-wide settings. Adjust escalation timing (1-1440 minutes) and answer detection mode.

/qr-targets

Manage multi-tier escalation targets. Add/remove users, user groups, or channels at any escalation level. Set priority order within each tier.

/qr-channel-config

Override workspace defaults for specific channels. Set custom escalation timings, answer modes, or disable escalation entirely for that channel.

/qr-channels

List all channels with custom configuration overrides. Shows escalation settings, answer detection mode, and whether escalation is enabled.

/qr-stats

Display comprehensive statistics. Shows questions asked/answered, response times, answer rates, and current escalation settings for monitoring.

/qr-test

Verify bot is running and view all user groups in workspace with their IDs. Helpful for configuration and troubleshooting.

/qr-test-escalation

Test your escalation configuration without waiting. Creates test question and immediately triggers escalation to verify targets.
