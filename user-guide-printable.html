What is Question Router?
Question Router is a Slack bot that automatically detects questions in your channels, tracks them, and ensures they get answered through intelligent escalation.

üöÄ Getting Started

How Questions Are Detected

The bot automatically identifies questions by looking for:

  • Messages ending with a question mark (?)
  • Messages starting with question words (How, What, When, Where, Why, Who)
  • Common help patterns ("Does anyone know...", "Can someone help...")
üí° Tip: Make sure your question is clear and specific. The bot works best with well-formed questions.

What Happens When You Ask a Question

  1. Detection: Bot detects your question and starts tracking it
  2. Waiting Period: Grace period allows someone to answer naturally
  3. First Escalation: If unanswered, support team is notified in the thread
  4. Second Escalation: If still unanswered, question posted to escalation channel
  5. Resolution: Once answered, escalation stops automatically

üí° Best Practices

When Asking Questions

  • Be specific and clear about what you need
  • Include context (what you've tried, error messages, etc.)
  • Ask in the appropriate channel for the topic
  • Wait for the escalation system to work - avoid reposting immediately

When Answering Questions

  • Reply in a thread to keep the channel organized
  • If you're not sure, say so - it helps others know to jump in
  • Mark questions resolved with ‚úÖ emoji (if in emoji_only or hybrid mode)
  • If a question is off-topic or unclear, suggest clarification

üÜò Troubleshooting

My question wasn't detected

  • Make sure your message ends with a question mark OR starts with a question word
  • Check if the bot has been invited to the channel
  • Verify the message is at least 10 characters long

Question was marked answered but shouldn't be

  • This can happen in thread_auto mode if someone replies but doesn't fully answer
  • Ask your admin to consider switching to hybrid mode for better control

Not getting escalation notifications

  • Check if you're in the configured user group for escalations
  • Verify notification settings for the escalation channel
  • Ask your admin to run /qr-config to check settings
