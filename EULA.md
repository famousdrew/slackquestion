## 3. Description of Service

Slack Question Router is a Slack application that:

- Automatically detects questions posted in monitored Slack channels
- Tracks unanswered questions and provides visual feedback
- Escalates unanswered questions to designated support teams or channels
- Provides statistics and analytics on question response rates
- Allows workspace administrators to configure escalation rules and timing

## 4. Data Collection and Privacy

### 4.1 Data We Collect

The Application collects and processes the following data from your Slack workspace:

**Message Data:**
- Message content from monitored channels (to detect questions)
- Message timestamps and thread information
- Reactions added to messages

**User Data:**
- User IDs and display names of users who post questions
- User IDs of users designated as escalation targets

**Workspace Data:**
- Workspace ID and name
- Channel IDs and names of monitored channels
- User group IDs and names used for escalations

**Configuration Data:**
- Escalation timing settings
- Answer detection mode preferences
- Configured escalation targets (users, groups, channels)

### 4.2 How We Use Your Data

We use the collected data exclusively to:

- Detect and track questions in your workspace
- Route unanswered questions to appropriate team members
- Generate statistics and analytics on question response performance
- Maintain application configuration and preferences

**We do NOT:**
- Share your data with third parties for marketing purposes
- Use your data to train AI models or for any purpose outside the Application's core functionality
- Sell your data to anyone
- Store email addresses or personally identifiable information beyond what's necessary for the Service

### 4.3 Data Retention

We maintain clear data retention policies to minimize data storage:

- **Answered questions:** Retained for 90 days, then automatically deleted
- **Dismissed questions:** Retained for 30 days, then automatically deleted
- **Unanswered questions:** Retained for 365 days, then automatically deleted
- **Workspace configuration:** Retained while you use the Service

These retention periods comply with GDPR Article 5 (data minimization and storage limitation) and can be configured by contacting support.

### 4.4 Data Security

We implement industry-standard security measures including:

- Encrypted database connections (PostgreSQL with SSL/TLS)
- Secure token storage for Slack API credentials
- Regular security audits and updates
- Access controls and authentication mechanisms
- Logging and monitoring for suspicious activity

### 4.5 Data Location

Your data is stored in secure cloud infrastructure. The specific location depends on your deployment configuration. For questions about data residency, please contact us.
