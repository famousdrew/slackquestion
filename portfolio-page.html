How It Works

Simple workflow, powerful results

1

Question Posted

User posts a question in any monitored Slack channel. The bot detects it instantly and saves it to the database.

2

First Escalation

After the configured time (default 2 minutes), if unanswered, the bot posts in the thread mentioning your support team.

3

Second Escalation

If still unanswered after the second threshold (default 4 minutes), the question is posted to your escalation channel for wider visibility.

4

Resolution

Once answered (via emoji reaction or thread reply depending on mode), escalation stops and the question is marked resolved.

Available Commands

Control your question router with simple slash commands

/qr-stats

Display comprehensive statistics about questions in your workspace. Shows total questions asked, answered, unanswered, average response time, and current escalation settings.

/qr-config

View and modify your workspace configuration. Adjust escalation timings (first and second escalation minutes) and change answer detection mode (emoji_only, thread_auto, or hybrid).

/qr-help

Get help and documentation directly in Slack. Shows available commands, configuration options, and how to use the question router effectively.

Usage Guide

Get started in minutes

üöÄ Getting Started

  • Install the Slack app in your workspace
  • Invite the bot to channels you want to monitor (e.g., /invite @unanswered_questions)
  • Configure your escalation settings using /qr-config
  • Set up your escalation channel (where second-level escalations are posted)
  • Define your support user group (who gets mentioned in first escalations)

‚öôÔ∏è Configuration

Set your environment variables or use the /qr-config command:

FIRST_ESCALATION_MINUTES=2 # Time until first escalation SECOND_ESCALATION_MINUTES=4 # Time until second escalation ESCALATION_USER_GROUP=t2 # User group to mention (@t2) ESCALATION_CHANNEL=C123456 # Channel ID for escalations

üéØ Answer Detection Modes

  • emoji_only - Questions require a checkmark emoji (‚úÖ or ‚úîÔ∏è) reaction to mark as answered. Most precise control.
  • thread_auto - Any reply in the thread automatically marks the question as answered and stops escalation.
  • hybrid - Thread replies pause escalation but don't mark as complete. Still requires emoji for final resolution.

üìà Monitoring

  • Run /qr-stats to see your team's performance metrics
  • Review average response times to identify bottlenecks
  • Check unanswered question counts to gauge support load
  • Monitor escalation patterns to optimize timing settings
  • Track daily/weekly trends to improve support coverage

‚ú® Best Practices

  • Start with conservative escalation times (2 and 4 minutes) and adjust based on your team's response patterns
  • Use emoji_only mode for high-stakes support channels where you need explicit confirmation
  • Use thread_auto mode for casual discussion channels where any response indicates engagement
  • Create a dedicated escalation channel (like #urgent-questions) with notifications enabled
  • Review stats weekly to identify peak question times and adjust team coverage
  • Train team members to use emoji reactions consistently

üîê Data & Privacy

All question data is stored securely in a PostgreSQL database. The bot only tracks message text, timestamps, and metadata necessary for escalation logic. No message content is shared with third parties.
